SoundTrap SWV Sensor Import

Overview

The SoundTrap SWV Sensor Import module extracts magnetometer and accelerometer data from SoundTrap SUD files (.swv streams) and uses a Mahony complementary sensor-fusion filter to compute heading, pitch and roll in real time. The computed orientation and the raw sensor values are stored to PAMGuard binary files and can be visualised on any time-base (TD) display.

The module is primarily intended for use with SoundTrap recorders that carry an inertial measurement unit (IMU), such as the SoundTrap 600 which uses an LSM303AGR combined accelerometer/magnetometer chip. SWV files contain six channels of 16-bit PCM data:

ChannelSignalUnits (after calibration)
0Accelerometer Xm/s²
1Accelerometer Ym/s²
2Accelerometer Zm/s²
3Magnetometer XµT
4Magnetometer YµT
5Magnetometer ZµT

Adding the Module

To add the module go to File → Add Modules → Sensors → SoundTrap SWV Sensor Import. The module will automatically subscribe to the SUD file stream produced by the main PAMGuard sound acquisition module. No additional acquisition configuration is required beyond selecting a SUD file source.

The module also requires the Binary Storage module in order to save orientation and raw sensor data to disk: File → Add Modules → Utilities → Binary File Storage.

Magnetometer Calibration

The magnetometer section allows correction of systematic biases introduced by nearby ferromagnetic material (hard-iron distortion) and scale asymmetries between axes (soft-iron distortion).

  • Hard-iron offsets X/Y/Z (µT) — constant offsets subtracted from each raw magnetometer reading before calibration. These represent fixed magnetic fields generated by nearby permanent magnets or magnetised components of the recorder housing. Default: 0.
  • Scale factors X/Y/Z — multiplicative correction applied after the hard-iron offset. Values close to 1.0 indicate a well-calibrated sensor. Default: 1.
  • LSB → µT factor — converts raw 16-bit ADC counts to micro-Tesla. For the LSM303AGR at the default ±50 gauss full-scale range the factory value is 0.15 µT/LSB.

Accelerometer Calibration

  • Offsets X/Y/Z — constant offsets subtracted from each raw accelerometer reading. Default: 0.
  • LSB → m/s² factor — converts raw 16-bit ADC counts to metres per second squared. For the LSM303AGR in high-resolution mode at ±2 g full-scale the factory value is 9.80665 × 10⁻³ m/s²/LSB (1 mg per LSB).

Sensor Fusion (Mahony Filter)

Because the SWV stream contains only a magnetometer and an accelerometer (no gyroscope), PAMGuard uses a Mahony complementary filter to estimate orientation. The filter maintains an orientation quaternion and corrects it each sample using the cross-product error between the measured gravity/field vectors and the vectors predicted from the current quaternion.

  • Proportional gain Kp — controls how aggressively the filter corrects orientation errors each sample. Higher values converge faster but may amplify noise. Typical range 1–10. Default: 2.0.
  • Integral gain Ki — removes steady-state bias in the error signal. Keep small to avoid instability. Default: 0.005.
  • Magnetic declination (°, +E) — the angle between magnetic north and true north at the deployment location. Positive values are East of true north. This is added to the computed magnetic heading to give a true heading. Enter the value for your deployment site (e.g. from NOAA's online calculator). Default: 0.
  • Max contiguous gap (ms) — when consecutive SUD files are processed the module checks whether the end of the previous file and the start of the next file are within this time gap. If they are, the filter quaternion is carried over rather than being reset, which avoids the heading/pitch/roll spike that would otherwise occur at the start of each file as the filter beds in. Default: 2000 ms.

Device Defaults

Click the Defaults ► button to load factory calibration values for a specific device type. Currently available:

  • ST600 (LSM303AGR) — SoundTrap 600 with the LSM303AGR sensor chip. Sets the accelerometer scale to 9.80665×10⁻³ m/s²/LSB (±2 g, high-resolution mode) and the magnetometer scale to 0.15 µT/LSB (±50 gauss range). All offsets are reset to zero. The magnetic declination is preserved.

Side Panel

In real-time processing mode a compact side panel is shown in the left-hand column of the PAMGuard main window. It displays the most recently computed heading, pitch and roll values, updated approximately four times per second.

Visualising Data on the Time-Base Display

The SudSensor module registers a data overlay for PAMGuard's Time Base Display. To add it, open or create a User Display, add a Time Base panel, then click the data-layer selector and choose SUD Sensor Data.

Two Y-axis options are available from the graph settings panel:

  • Orientation (°) — plots heading (0–360°), pitch (±90°) and roll (±180°) as three continuous lines.
  • Raw Sensor — plots the six raw (calibrated) sensor channels: magnetometer X/Y/Z (µT) and accelerometer X/Y/Z (m/s²).

Line Colour Settings

Each of the nine plotted lines has an independently configurable colour and enable/disable toggle. Click the settings tab (labelled Sensor Lines) on the right-hand side of any Time Base Display panel that shows SudSensor data. The panel is divided into three sections:

  • Orientation — heading, pitch and roll (enabled by default).
  • Magnetometer — mag X, Y, Z (disabled by default).
  • Accelerometer — accel X, Y, Z (disabled by default).

Data Map / Datagram

The module provides a datagram that is shown on PAMGuard's data map. The datagram displays average heading, pitch and roll across three colour bands, giving a quick visual overview of the recorder's orientation throughout a deployment.

Binary Storage

When the Binary Storage module is present, each sensor sample is saved as a binary data unit containing:

  • UTC timestamp
  • Calibrated magnetometer X, Y, Z (µT)
  • Calibrated accelerometer X, Y, Z (m/s²)
  • Fused heading (°), pitch (°) and roll (°)

Binary files are named using the standard PAMGuard convention: SoundTrap_Sensor_[name]_SudSensorData_YYYYMMDD_HHMMSS.pgdf.

File Continuity and Filter Bedding-In

The Mahony filter requires several seconds of data before its heading estimate stabilises (the "bedding-in" period). When SoundTrap recorders are deployed they typically produce a series of consecutive SUD files. If the gap between the end of one file and the start of the next is within the Max contiguous gap threshold (default 2 s), the module carries the existing filter state forward rather than resetting it. This eliminates the heading/pitch/roll spike that would otherwise appear at the start of every file.

If a gap larger than the threshold is detected (or at the very start of processing) the filter is reset and a new bedding-in period begins.

Troubleshooting

SymptomLikely causeSuggested action
No data appears in the TD display SUD files do not contain a SWV stream, or the SWV stream has a different channel layout Check that the SoundTrap was configured to record SWV data. Verify the channel layout matches channels 0–2 = accelerometer, 3–5 = magnetometer.
Heading shows repeated spikes at the start of each file Max contiguous gap is too small, or files have genuine time gaps Increase Max contiguous gap in the settings dialog, or check that the SUD files are truly contiguous.
Heading drifts or is consistently offset from true north Magnetic declination not set, or hard-iron calibration needed Enter the site magnetic declination in the settings dialog. If the offset persists across different headings, perform a hard-iron calibration and enter the offset values.
Orientation lines do not appear in the TD display Lines are disabled in the Sensor Lines settings pane Open the Sensor Lines tab and click Show for the required lines.
Binary files are not being saved Binary Storage module not added, or binary storage is not enabled for this data block Add the Binary File Storage module and check that SUD Sensor Data is ticked in the binary storage settings.
